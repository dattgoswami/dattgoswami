# Datt Goswami

Engineer building Rust LLM infrastructure, agent tooling, and continual-learning systems for coding agents.

Currently building:

- [`cl-agent`](https://github.com/dattgoswami/cl-agent) — continual-learning substrate for coding agents
- [`axon`](https://github.com/dattgoswami/axon) — async Rust LLM inference server
- **Ferrum** — agent tooling, memory, evals, MCP, tracing, and platform infrastructure

Docs: [Ferrum Stack](https://drive.google.com/file/d/15N13R1c0gWQDTTNzfVQruilHJWlRwneQ/view?usp=sharing) · [Ferrum Platform Poster](https://drive.google.com/file/d/1yD4BJmNPHm3lDecZRchCPDTh9vz-uBc9/view?usp=sharing)

## Featured Work

| Project | What it is | Why it matters |
|---|---|---|
| [`cl-agent`](https://github.com/dattgoswami/cl-agent) | Continual-learning substrate for coding agents | Episode replay, skill distillation, SOAR-style search, SFT datasets, and promotion-gated evals — [paper](https://github.com/dattgoswami/cl-agent/blob/main/paper/cl_agent_paper_pub.pdf) |
| [`axon`](https://github.com/dattgoswami/axon) | Async Rust LLM inference server | Candle-based local inference with dynamic batching, SSE streaming, Metal/CUDA/CPU fallback, and TurboQuant-style KV-cache compression |
| [`ferrum-core`](https://github.com/dattgoswami/ferrum-core) | Shared Rust foundation for Ferrum services | Unified errors, fixed-schema JSON logging, and OTLP tracing without HTTP-framework lock-in |
| [`PersonalKB`](https://github.com/dattgoswami/PersonalKB) | Offline-first local RAG CLI | Hybrid retrieval over books and papers with Ollama, BM42 sparse vectors, Qdrant RRF, and cited answers |
| [`ferrum-mcp`](https://github.com/dattgoswami/ferrum-mcp) | Rust MCP server for AI agents | Typed agent access to crypto prices, DeFi yield/risk, Solana wallet state, and semantic memory over stdio/HTTP |
| [`forge-core`](https://github.com/dattgoswami/forge-core) | Framework-agnostic SWE-agent core | Shared tools, schemas, guardrails, Docker sandbox, evals, and adapters for major agent SDKs |
